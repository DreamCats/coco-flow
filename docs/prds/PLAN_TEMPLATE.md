# Plan 模板

> `Design` 负责讲清楚方案，`Plan` 负责讲清楚怎么做。

## 目标

- 这次执行要完成什么
- 与 `design.md` 的对应关系是什么

## 前置依赖

- 依赖哪个仓库先改
- 是否需要先升级依赖版本
- 是否依赖 IDL、配置或发布窗口

## 顺序与并行关系

- 哪些步骤必须串行
- 哪些步骤可以并行
- 哪一步是关键路径

## 验证计划

- 最小编译/测试命令
- 需要重点人工确认的行为
- 是否要分别验证实验命中与未命中

## 回滚与兜底

- 如果某一步失败，先回滚哪部分
- 是否允许只发布上游或只发布下游
- 是否需要临时降级开关
