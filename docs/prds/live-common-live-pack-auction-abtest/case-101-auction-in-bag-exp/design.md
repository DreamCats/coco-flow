# Design

## 核心改造点

- 通过实验控制“竞拍进购物袋”能力是否生效
- 实验未命中时，购物袋保持原有普通商品路径
- 实验命中时，购物袋主链路与辅助返回口径一致

## 仓库职责

- `oec/live_common/abtest`
  负责定义实验字段和对外暴露实验结构

- `ttec/live_pack`
  负责消费实验字段，并在购物袋入口、辅助返回和结果写回层统一接线

## 仓库依赖关系

- 依赖方向：`live_pack -> oec/live_common/abtest`
- 不是双向依赖
- 技术顺序：先改上游实验字段，再升级 `live_pack` 依赖，最后改 `live_pack` 业务逻辑

## 影响范围与边界

- 影响范围：购物袋竞拍入口、相关辅助返回、实验外结果隔离
- 不影响：竞拍样式、竞拍 banner 内容、普通购物袋主路径
- 风险点：如果只改主入口，不改辅助返回，会出现实验外残留竞拍数据

## 人力评估

- 复杂度：中
- 预估人力：1 到 2 人天
- 更偏功能开关接线和返回口径收敛
