# Design

## 核心改造点

- 通过实验控制普通竞拍成交后是否展示“成交成功”
- 讲解卡、购物袋和其他返回路径都使用同一状态口径
- 实验未命中时保持当前状态表达

## 仓库职责

- `oec/live_common/abtest`
  负责定义成交态实验字段

- `ttec/live_pack`
  负责把实验字段接到普通竞拍状态判定和各入口返回逻辑

## 仓库依赖关系

- 依赖方向：`live_pack -> oec/live_common/abtest`
- 先更新实验字段，再升级依赖，最后统一下游状态表达
- 多条返回路径共享同一实验口径，不允许各自独立演化

## 影响范围与边界

- 影响范围：普通竞拍成交态表达
- 不影响：惊喜盲盒、按钮文案、TCC 文案
- 风险点：如果只改讲解卡，不改购物袋或其他返回路径，会造成同一竞拍多入口状态不一致

## 人力评估

- 复杂度：中
- 预估人力：1 到 2 人天
- 适合作为多仓“实验字段 + 多链路状态统一” case
