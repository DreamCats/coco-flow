# PRD

## 需求名称

竞拍讲解卡成交后展示独立 Success 态

## 背景

当前竞拍成交后，不同入口下的讲解卡状态表达并不稳定。对用户来说，同样都是“竞拍已成交”，有时会看到更偏“成功”的状态表达，有时又会被归入更笼统的“结束”状态。

这会导致用户对当前拍卖结果的理解不够直接。

## 目标

把竞拍讲解卡的成交态收敛为统一的“成交成功”表达，用来明确区分“已成交”和“已结束/流拍”。

## 范围

- 仅处理普通竞拍讲解卡
- 仅处理直播间讲解卡场景
- 不改购物袋竞拍列表
- 不改惊喜盲盒竞拍

## 非目标

- 不新增 TCC 配置
- 不改按钮文案内容
- 不改成交后的跳转和 schema

## 用户故事

作为直播间用户，当竞拍已成交时，我希望讲解卡展示明确的“成交成功”状态，而不是和“已结束”混在一起，这样我能更快理解当前拍卖结果。

## 验收标准

1. 当普通竞拍已成交时，讲解卡展示明确的“成交成功”状态
2. “成交成功”与“已结束/流拍”在用户可见表达上要能区分
3. 已结束的竞拍仍然保持原有结束态表达
4. 只改讲解卡场景，不影响购物袋竞拍列表
