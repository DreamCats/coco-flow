# Plan

## 目标

- 让购物袋竞拍 banner 与竞拍商品结果绑定
- 保证不同购物袋接口返回一致

## 前置依赖

- 无跨仓依赖
- 不涉及 banner 内容改造

## 执行切片

### Slice 1

- 改动范围：竞拍列表结果层
- 主要文件/模块：banner 生成条件
- 预期产出：无竞拍商品时不生成 banner
- 风险：若条件判断位置不对，可能遗漏边界场景

### Slice 2

- 改动范围：购物袋数据接口
- 主要文件/模块：data handler 返回写回
- 预期产出：无竞拍商品时不回传 banner
- 风险：若只改结果层，不改接口层，仍可能写回旧字段

### Slice 3

- 改动范围：购物袋组装接口
- 主要文件/模块：assemble handler 返回写回
- 预期产出：不同接口 banner 口径一致
- 风险：若漏改，会造成接口间表现不一致

## 顺序与并行关系

- Slice 1 必须优先
- Slice 2 和 Slice 3 可并行
- 关键路径是 banner 生成条件的统一

## 验证计划

- 最小验证：`go test ./entities/converters/auction_converters ./handlers`
- 重点确认：无竞拍商品时不返回 banner
- 回归确认：有竞拍商品时 banner 行为不变

## 回滚与兜底

- 若接口回归异常，可先回滚对应 handler 写回
- 若根因在生成条件，再回滚列表结果层
- 不需要额外降级开关
