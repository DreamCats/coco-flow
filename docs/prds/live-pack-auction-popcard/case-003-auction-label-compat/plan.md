# Plan

## 目标

- 补齐普通竞拍讲解卡的拍卖标签
- 保证所有返回路径标签样式一致

## 前置依赖

- 无跨仓依赖
- 复用现有标签配置，不新增配置结构

## 执行切片

### Slice 1

- 改动范围：补齐缺失标签的返回路径
- 主要文件/模块：兼容返回结构组装逻辑
- 预期产出：拍卖标签在缺失路径下也能稳定返回
- 风险：若直接复制逻辑，后续容易再次分叉

### Slice 2

- 改动范围：一致性核对
- 主要文件/模块：主路径与兼容路径标签对齐
- 预期产出：文案、颜色、图标完全一致
- 风险：若只对齐文案，不对齐颜色和图标，仍会有视觉差异

## 顺序与并行关系

- 先补齐缺失路径，再做一致性核对
- 两步不建议并行
- 关键路径是标签结构与现有口径保持一致

## 验证计划

- 最小验证：`go test ./entities/loaders/product_loaders`
- 重点确认：拍卖标签在原缺失路径下已返回
- 回归确认：现有 placement label 和 promotion label 不受影响

## 回滚与兜底

- 若标签结构不兼容，先回滚补齐逻辑
- 保留当前已正确的主路径行为
- 不需要额外降级开关
