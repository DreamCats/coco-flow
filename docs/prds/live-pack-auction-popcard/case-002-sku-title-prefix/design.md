# Design

## 核心改造点

- 多规格竞拍商品在标题中补充当前参拍规格名
- 单规格商品保持原有标题
- 不同返回路径下的标题规则统一

## 系统职责

- 标题组装层：决定什么时候展示 `#规格名 商品标题`
- 兼容返回层：保证不同返回路径下标题规则一致
- 商品数据层：继续复用现有规格信息，不新增取数来源

## 依赖关系

- 单仓改造，无跨仓依赖
- 标题规则依赖现有规格信息和当前竞拍规格识别结果

## 影响范围与边界

- 影响范围：普通竞拍讲解卡标题
- 不影响：图片、价格、schema、购物袋
- 风险点：如果误伤单规格商品，会造成标题冗余

## 人力评估

- 复杂度：低
- 预估人力：0.5 到 1 人天
- 适合作为单仓标题类 case
