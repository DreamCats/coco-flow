# Plan

## 目标

- 统一延时竞拍预热态的双按钮文案类型
- 讲解卡、购物袋与其他返回路径保持一致

## 前置依赖

- 无跨仓依赖
- 依赖现有延时竞拍判定结果可复用

## 执行切片

### Slice 1

- 改动范围：讲解卡主路径
- 主要文件/模块：双按钮文案类型判定
- 预期产出：延时竞拍预热态使用专属 init 文案类型
- 风险：若误伤普通竞拍，会导致普通预热态按钮异常

### Slice 2

- 改动范围：购物袋路径
- 主要文件/模块：bag 双按钮文案类型映射
- 预期产出：购物袋与讲解卡口径一致
- 风险：若 helper 或调用方未同步，bag 会继续沿用旧口径

### Slice 3

- 改动范围：兼容返回路径
- 主要文件/模块：旧路径双按钮文案类型逻辑
- 预期产出：所有返回路径一致
- 风险：漏改会造成多入口表达分裂

## 顺序与并行关系

- 先明确统一规则
- Slice 1 和 Slice 2 可并行
- Slice 3 在规则稳定后再补齐
- 关键路径是规则是否同时覆盖主路径和 bag

## 验证计划

- 最小验证：`go test ./entities/converters/auction_converters ./entities/loaders/product_loaders`
- 重点确认：延时竞拍预热态按钮使用专属口径
- 回归确认：普通竞拍预热态、成交态、结束态不变

## 回滚与兜底

- 若 bag 或兼容路径出问题，可单独回滚对应切片
- 若核心规则有误，整体回滚统一口径改动
- 不需要额外降级开关
