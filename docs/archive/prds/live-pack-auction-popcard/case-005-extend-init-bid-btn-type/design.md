# Design

## 核心改造点

- 延时竞拍预热态的双按钮文案改为使用延时竞拍专属口径
- 普通竞拍预热态保持原有按钮口径
- 讲解卡、购物袋及其他返回路径保持一致

## 系统职责

- 文案类型判定层：统一预热态按钮文案类型
- 讲解卡表达层：输出延时竞拍专属按钮表达
- 购物袋与其他返回路径：沿用同一规则，避免入口分叉

## 依赖关系

- 单仓改造，无跨仓依赖
- 同仓内多个入口共享同一套“延时竞拍预热态”判定口径

## 影响范围与边界

- 影响范围：双按钮场景下的延时竞拍预热态表达
- 不影响：单按钮场景、成交态、结束态、惊喜盲盒
- 风险点：如果部分入口未同步，会出现标题和按钮、讲解卡和购物袋口径不一致

## 人力评估

- 复杂度：中低
- 预估人力：1 人天
- 适合作为单仓多链路一致性 case
