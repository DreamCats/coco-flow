# PRD

## 需求名称

延时竞拍双按钮预热态使用专属 Init 按钮文案类型

## 背景

当前延时竞拍在预热阶段已经有自己独立的标题表达，但双按钮场景下的按钮文案仍然沿用普通竞拍预热态。

这会导致同一张卡片上，标题和按钮表达不一致，削弱延时竞拍的专属感知。

## 目标

当延时竞拍处于预热态时，双按钮场景下的按钮文案也使用延时竞拍专属的预热态表达。

## 改动点

1. 直播间讲解卡对齐
2. 购物袋竞拍卡对齐
3. 其他返回路径下的讲解卡也要对齐

## 范围

- 仅 regular auction
- 仅双按钮场景下的 `BidBtnTextType`
- 不改 `AuctionTextType`

## 非目标

- 不改按钮具体文案内容
- 不改 SurpriseSet
- 不改单按钮场景

## 验收标准

1. 延时竞拍预热态下，双按钮使用延时竞拍专属预热态文案
2. 普通竞拍预热态仍保持原按钮文案
3. 竞价中、成交、结束态不变
4. 讲解卡、购物袋和其他返回路径表达一致
