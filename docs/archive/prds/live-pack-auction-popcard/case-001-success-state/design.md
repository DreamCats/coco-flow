# Design

## 核心改造点

- 把普通竞拍成交后的用户可见状态统一收敛为“成交成功”
- 明确区分“已成交”和“已结束/流拍”
- 讲解卡不同返回路径保持一致

## 系统职责

- 状态判定层：统一定义普通竞拍成交后的状态口径
- 讲解卡返回层：保证不同返回路径下状态一致
- 购物袋与惊喜盲盒：不在本次改造范围内

## 依赖关系

- 单仓改造，无跨仓依赖
- 同一仓库内存在多条讲解卡返回路径，需要共享同一状态口径

## 影响范围与边界

- 影响范围：普通竞拍讲解卡状态表达
- 不影响：购物袋竞拍、惊喜盲盒竞拍、按钮文案、TCC 文案
- 风险点：如果只改一条返回路径，会造成同一竞拍在不同入口下状态不一致

## 人力评估

- 复杂度：低
- 预估人力：0.5 人天
- 适合作为单仓低风险对照组
