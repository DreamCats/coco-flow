# Design

## 核心改造点

- 补齐普通竞拍讲解卡的拍卖标签表达
- 保证不同返回路径下标签一致
- 当前已正确展示标签的路径保持不变

## 系统职责

- 标签组装层：统一产出拍卖标签
- 兼容返回层：补齐当前缺失标签的路径
- 文案与样式来源：继续沿用现有配置来源，不扩展配置模型

## 依赖关系

- 单仓改造，无跨仓依赖
- 依赖同一套标签配置口径，但不新增新的外部依赖

## 影响范围与边界

- 影响范围：普通竞拍讲解卡左上角拍卖标签
- 不影响：placement label、promotion label、购物袋
- 风险点：若不同路径自行拼标签，后续容易再次分叉

## 人力评估

- 复杂度：低
- 预估人力：0.5 人天
- 适合作为一致性补齐类 case
