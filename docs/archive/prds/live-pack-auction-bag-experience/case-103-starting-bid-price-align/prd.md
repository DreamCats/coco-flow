# PRD

## 需求名称

购物袋 regular auction 预热态起拍价与讲解卡保持一致

## 背景

- 用户在讲解卡和购物袋里看到同一场 regular auction 时，预热态起拍价口径不一致，容易让人误以为两个入口展示了不同价格
- 产品希望用户在不同入口之间切换时，看到的是同一套起拍价表达

## 目标

- regular auction 在购物袋预热态展示的起拍价与讲解卡保持一致
- 起拍价来源应稳定，不再依赖容易漂移的旧价格字段
- 只修正起拍价，不扩大到其他价格项

## 范围

- 仅购物袋 regular auction 的 `start_bid_price`
- 仅预热态起拍价口径
- 仅购物袋与讲解卡之间的一致性对齐

## 非目标

- 不改当前最高价
- 不改下一口价
- 不改自定义出价
- 不改 surprise set

## 用户故事

作为同时会看讲解卡和购物袋的用户，我希望两个入口展示的预热态起拍价一致，这样我不会怀疑商品价格是不是发生了变化。

## 验收标准

1. shopping bag 中 regular auction 的 `start_bid_price` 与讲解卡使用同一套起拍价口径。
2. 起拍价来自当前竞拍配置中的起拍价字段，而不是旧的商品价格字段。
3. ongoing 等其他状态的价格展示保持现有逻辑不变。
4. surprise set 不受影响。
