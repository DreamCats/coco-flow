# Design

## 核心改造点

- 这是购物袋首屏落点修正，不是 tab 体系改造
- 默认 tab 的兜底逻辑需要基于当前页实际商品结果判断
- assemble 和 data 两条返回路径都要一致处理

## 系统职责

- `live_pack` handler：在最终响应组装阶段决定默认 tab 兜底值
- `AuctionListDto`：继续提供竞拍商品结果
- BagTabs：继续按现有逻辑生成，不在本次改造范围

## 依赖关系

- 主责任仓库是 `ttec/live_pack`
- 不依赖外部协议或配置变更
- 不需要联动 BFF 或前端

## 影响范围与边界

- 影响范围：购物袋响应中的 `DefaultBagTab`
- 不影响：BagTabs 结构、竞拍商品列表、banner
- 风险点：若只改一条返回路径，会出现接口间行为不一致

## 人力评估

- 复杂度：低
- 预估人力：0.5 人天
- 适合作为购物袋体验类基线 case
