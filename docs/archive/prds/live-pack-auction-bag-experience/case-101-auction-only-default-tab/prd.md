# PRD

## 需求名称

购物袋当前页只有竞拍商品时默认打开竞拍 Tab

## 背景

- 用户打开购物袋时，如果当前页只有竞拍商品，却默认落在 Buy Now，会让首屏内容和默认 tab 语义不一致
- 这种不一致会增加用户理解成本，也会让竞拍商品入口被弱化

## 目标

- 当购物袋当前页只有竞拍商品时，默认打开竞拍 tab
- 当购物袋当前页还有 Buy Now 商品时，维持原有默认打开 Buy Now 的行为
- 默认 tab 的选择应与当前页真实内容一致

## 范围

- 仅购物袋默认 tab 行为
- 覆盖 assemble 和 data 两条返回路径
- 仅基于当前页实际商品数据判断

## 非目标

- 不改 BagTabs 顺序
- 不改分页逻辑
- 不改 banner 展示逻辑
- 不改竞拍商品排序

## 用户故事

作为打开购物袋的用户，如果当前页看到的都是竞拍商品，我希望页面直接停留在竞拍 tab，而不是先落到一个没有对应内容的 Buy Now tab。

## 验收标准

1. 当前页只有竞拍商品、没有 Buy Now 商品时，默认打开竞拍 tab。
2. 当前页同时存在 Buy Now 商品时，默认仍保持 Buy Now。
3. assemble 和 data 两条返回路径行为一致。
4. 不因为这个改动改变已有 BagTabs 的生成逻辑。
