# Design

## 核心改造点

- 这是价格展示优化，不是价格计算逻辑改造
- 仅在讲解卡 converter 层对实验命中的整数展示做轻量裁剪
- 不下沉到通用格式化层，避免误伤其他价格链路

## 系统职责

- `live_pack`：在 regular / surprise set converter 层处理价格展示裁剪
- 通用价格格式化层：继续保持不变
- 购物袋等其他价格消费方：不在本次范围

## 依赖关系

- 主责任仓库是 `ttec/live_pack`
- 依赖现有实验字段
- 不需要新增协议或价格字段

## 影响范围与边界

- 影响范围：讲解卡整数价格的展示字符串
- 不影响：金额值本身、非整数金额、购物袋价格
- 风险点：若裁剪位置过低，会把其他价格链路一起改掉

## 人力评估

- 复杂度：低
- 预估人力：0.5 人天
- 适合作为价格展示类基线 case
