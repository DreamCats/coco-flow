# Design

## 核心改造点

- 把竞拍爽感实验收敛为讲解卡表达层改造，而不是改竞拍主流程
- regular auction 和 surprise set 分别按自己的状态枚举覆盖目标文案
- 实验未命中时必须完整保留当前线上逻辑

## 系统职责

- `live_pack`：消费已有实验字段，按状态改写讲解卡文案
- 竞拍状态层：继续沿用现有状态判断，不新增新状态
- 购物袋、BFF、协议层：不在本次范围

## 依赖关系

- 主责任仓库是 `ttec/live_pack`
- 依赖现有实验字段可在 `RequestContext` 中读取
- 不需要新增协议或前端字段

## 影响范围与边界

- 影响范围：regular auction / surprise set 的 `AuctionText`
- 不影响：按钮、schema、价格格式、购物袋
- 风险点：如果状态映射不完整，可能出现部分状态文案新旧混用

## 人力评估

- 复杂度：低到中
- 预估人力：0.5 到 1 人天
- 适合作为“实验驱动的表达层变化”基线 case
