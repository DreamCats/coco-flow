# PRD

## 需求名称

购物袋返回默认打开 Tab

## 背景

当前购物袋在同时存在“普通商品”和“竞拍商品”的情况下，前端希望知道默认应打开哪个 Tab，避免首屏展示与服务端推荐逻辑不一致。

现状是服务端内部已经有默认 Tab 口径，但 API 没有稳定透出给前端，导致前端只能自行兜底。

## 目标

在购物袋接口中新增“默认打开 Tab”字段，让前端能够直接按服务端口径落地默认页签。

## 范围

- 仅购物袋接口
- 仅新增 API 返回字段
- 不调整现有商品和竞拍列表内容

## 非目标

- 不修改默认 Tab 判定规则
- 不调整购物袋排序逻辑
- 不改客户端交互样式

## 验收标准

1. 购物袋响应中新增默认打开 Tab 字段
2. 当前已有默认 Tab 口径保持不变，只是向前端稳定透出
3. 不影响现有商品列表和竞拍列表内容
