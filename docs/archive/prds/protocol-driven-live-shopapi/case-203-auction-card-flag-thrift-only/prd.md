# PRD

## 需求名称

竞拍讲解卡新增内部判断标记

## 背景

当前服务端希望在竞拍讲解卡链路中增加一个新的内部判断标记，用于支持服务端分流、兼容处理或埋点判断。

该标记仅用于服务内部逻辑，不需要直接返回给前端。

## 目标

为竞拍讲解卡链路新增一个可稳定透传的内部判断标记，供上层服务在服务端逻辑中直接使用。

## 范围

- 仅竞拍讲解卡链路
- 仅服务间字段透传
- 不新增前端 API 字段

## 非目标

- 不改动客户端返回结构
- 不新增前端展示逻辑
- 不修改竞拍主流程规则

## 验收标准

1. 服务端可以稳定拿到新的内部判断标记
2. 该标记可被上层服务用于内部逻辑判断
3. 不影响当前前端 API 返回结构
