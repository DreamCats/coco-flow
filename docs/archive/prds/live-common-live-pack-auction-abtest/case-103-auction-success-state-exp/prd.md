# PRD

## 需求名称

增加竞拍成交态实验，统一讲解卡与购物袋的 success 表达

## 背景

普通竞拍成交后，希望通过实验观察“成交成功”这种更明确的状态表达，是否比笼统的“结束”表达更利于用户理解。

但这个实验如果只改一个入口，就会导致同一个竞拍在不同位置看到的状态不一致。

## 目标

新增成交态实验。命中实验时，普通竞拍在不同入口下都统一展示“成交成功”表达；未命中时，保持当前口径。

## 改动点

1. 增加实验开关
2. 直播间讲解卡按实验控制成交态表达
3. 购物袋和其他返回路径也按相同口径控制成交态表达

## 范围

- 普通竞拍
- 直播间讲解卡
- 购物袋竞拍
- 其他竞拍返回路径

## 非目标

- 不改按钮文案
- 不改 TCC
- 不改 SurpriseSet

## 验收标准

1. 实验命中时，普通竞拍成交后统一展示“成交成功”
2. 实验未命中时，保持原有状态表达
3. 讲解卡、购物袋和其他返回路径表达一致
4. 已结束但未成交的竞拍仍保持原有结束态表达
