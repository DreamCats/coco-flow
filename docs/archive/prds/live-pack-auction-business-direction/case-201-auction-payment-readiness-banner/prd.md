# PRD

## 需求名称

购物袋竞拍 Tab 展示地址/支付准备状态提示

## 背景

- 竞拍商品进入购物袋后，用户可能在购物袋里才准备参与出价
- 如果用户未签署竞拍协议、未补齐收货地址或未设置支付方式，出价前会遇到额外阻断
- 这些准备项只依赖用户状态，不应该强依赖当前是否有 pin 竞拍卡

## 目标

- 当购物袋存在竞拍商品时，在竞拍 tab 展示必要的准备状态提示
- 未签约时提示用户先完成竞拍协议
- 已签约但缺地址或支付方式时，提示用户补齐对应信息
- 即使当前没有 pin 竞拍商品，只要竞拍 tab 有商品，也能展示准备提示

## 范围

- 仅购物袋竞拍 tab
- 覆盖 list/assemble 场景
- 提示内容包括竞拍协议、地址、支付方式
- schema 跳转复用现有竞拍协议和地址支付配置

## 非目标

- 不改竞拍协议内容
- 不改地址或支付页面
- 不改竞拍商品排序
- 不新增前端协议字段类型
- 不改变 pin 卡展示逻辑

## 用户故事

作为想参与竞拍的用户，我希望在购物袋竞拍 tab 里提前看到自己还缺哪些准备项，这样我能在真正出价前完成协议、地址和支付设置。

## 验收标准

1. 购物袋存在竞拍商品且用户未签约时，展示协议签署提示，并能跳转到协议签署页。
2. 用户已签约但缺地址时，展示补地址提示，并能跳转到地址/支付准备页。
3. 用户已签约但缺支付方式时，展示补支付提示，并能跳转到地址/支付准备页。
4. 用户协议、地址、支付方式都已完成时，不展示准备提示。
5. 当前无 pin 竞拍商品但竞拍 tab 有商品时，仍能展示准备提示。
