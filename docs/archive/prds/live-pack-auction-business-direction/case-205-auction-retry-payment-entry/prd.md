# PRD

## 需求名称

竞拍支付失败后提供重试支付入口

## 背景

- 用户竞拍成功后，可能因为地址、支付方式或支付链路异常导致支付失败
- 如果支付失败后只能依赖外部通知或订单页入口，用户回到支付链路的路径较长
- 竞拍卡和购物袋竞拍卡是用户继续处理竞拍订单的高频入口

## 目标

- 当竞拍订单处于支付失败可重试状态时，在竞拍卡展示可重试支付入口
- regular auction 和 surprise set 都支持重试支付
- schema 携带必要的 room、auction、config、term 信息
- 未命中实验或状态不满足时保持现有完成态展示

## 范围

- 覆盖竞拍讲解卡
- 覆盖购物袋竞拍卡
- 覆盖 regular auction 和 surprise set
- 只处理支付失败可重试场景

## 非目标

- 不改支付重试页
- 不改订单状态机
- 不改竞拍成功判定
- 不新增支付 RPC
- 不处理非竞拍订单

## 用户故事

作为中拍但支付失败的用户，我希望能从竞拍卡直接回到支付重试流程，这样我不用离开直播间去订单列表里查找失败订单。

## 验收标准

1. 命中实验且支付失败可重试时，竞拍卡下发重试支付入口。
2. 重试支付 schema 能带上 room_id、auction_id、auction_config_id 和协议版本等必要参数。
3. regular auction 和 surprise set 都支持该入口。
4. 非支付失败状态不展示重试入口。
5. 未命中实验时保持现有完成态展示。
